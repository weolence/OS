Требования к 1.6:

* Реализация в виде библиотеки
* Автоматическая сборка
* Обработка всех ошибок
* Отсутствие магических контстант
* CFLAGS = -Wall -Wextra -Werror -fsanitize=address -fsanitize=leak -g
* Выделенные ресурсы должны быть освобождены
* Разделение сложных функций на понятные части: конкретная функция должна делать конкретное действие
* При необходимости в коде должны быть понятные комментарии на английском, объясняющие логику работы
* Разные типы ошибок, а не просто thread_success \ thread_fail
* Структура библиотеки -- инклюды в `include`, сорсы в `src`, тесты в `tests`

* Функциональность:
create
exit
self
equal
join
detach
cancel
testcancel
cleanup_push | pop
